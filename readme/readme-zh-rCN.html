概览

规则 是 Dir 的核心内容。每一条规则 都指向了一个目录或文件。

注： Dir 暂只支持清理SD卡目录，请不要提交诸如 /system 或者 /data 需要 超级用户权限 才能修改的内容

哪些路径可以被作为规则？

以下文件（文件夹）可被作为规则。

  • 用户不需要，如 广告目录
  • 临时文件
  • 缓存文件
  • 日志文件

以下文件（文件夹）不应作为规则。

  • 用户的私有内容，如 照片
  • 系统目录，如LOST.DIR
  • 会导致系统或应用程序无法运行的内容

请遵守上述要求。

配置

详细解释规则的各条配置信息，您需要详细阅读，才可以使您的规则通过审核。

路径

必填

该规则的路径，可以是文件、文件夹。

注意

  • 您不可以填写路径中涉及 可变 目录或文件夹，它必须是在绝大多数设备上可被直接使用的，不受例如登录账号不同的约束。
  • 该路径不得以 您的SD卡根目录（机身存储路径） 开头。例如：您选取的路径 /sdcard/A 指 SD卡/A，您就需要填写 /A 或 A ，不得填入 /sdcard/A。

举例

  • /ABCDE/

标题

必填

该规则对用户可见的标题

注意

  • 请以最清晰简洁的方式写标题，且不得带有敏感词等违法词汇

举例

  • XXX垃圾文件

包名

可选（若勾选 需要卸载才视为垃圾 则必填）

与该规则关联的应用包名，也会作为图标显示

注意

  • 该路径必须仅由该应用产生的。若该路径是一些使用了某组件（SDK）产生的，则不可以填入包名。

默认不删除

勾选

当扫描到该路径时，默认不勾选 清除 复选框

何时勾选 该路径较为重要，需要用户谨慎处理。例如：下载文件

默认不替换

勾选

当扫描到该目录时，默认不勾选 替换 复选框

何时勾选 该文件可被删除，但替换后会出现问题。例如：缓存 以下不需要勾选：

  • 广告文件/文件夹
  • 日志文件/文件夹（慎重考虑）
  • 和广告有关等文件/文件夹（如自动默静下载的广告APK等）

需要卸载才视为垃圾

勾选

此规则仅当对应 包名 不存在才可见

何时勾选

此路径在对应应用已安装时较为重要，不得删除。但对应应用卸载后属于残留垃圾。

该路径是文件，不是文件夹

勾选

Dir 默认您的规则目标是一个目录

何时勾选

若您指向的路径是一个文件，则应该勾选该复选框

谨慎清理

勾选

提示用户谨慎清理这个文件（文件夹），通常用于用户数据等场合

谨慎替换

勾选

提示用户谨慎替换，通常用于缓存等场合

模板

提交工具1.15引入了 模板 概念，您可以根据自己规则的分类选择相应模板：

  • 缓存
  • 日志
  • 广告
  • 卸载残留（大多数是应用存储在SD卡的配置文件等）
  • 用户数据（例如下载等用户关键文件）

注意：

  • 若您的规则同时可使用多个模板，那么优先级依次是： 用户数据、卸载残留、广告、缓存、日志。如某规则可使用 卸载残留 和 用户数据， 那么选择 用户数据。

流程

您需要按照以下流程来正确提交规则：

  1. 填写规则详细配置（详见上文）
  2. 提交规则
  3. 等待管理员审核。管理员会严格按照上文所述约定和自己的见解来判断是否采纳您的规则。若被驳回，可能会说明原因。请关注我的规则列表和邮件。

注意：您的规则除非被驳回，否则无法撤回或修改。请确保它没有问题。如果出现严重问题，请邮件联系开发者帮您人工撤回。

最佳实践

这里展示一些规则示例：

  • 某应用广告文件夹
标题：XXX广告
模板：广告
默认不替换：否（这可以在一定程度上去除该广告）
默认不删除：否
谨慎清理：否
谨慎替换：否
  • 某视频软件缓存
标题：XXX视频缓存
模板：缓存
默认不替换：是（替换后可能导致视频等无法播放/查看）
默认不删除：否
谨慎清理：否
谨慎替换：是
  • 某软件日志
标题：XX日志
模板：日志
默认不替换：是（酌情勾选）
默认不删除：否
谨慎清理：否
谨慎替换：否

错误详解

  • 规则已存在

    该目录 已存在与 规则列表（用户可从Dir主程序获取到的规则）、审核列表（包括 待审核及已驳回（未重新提交）。 为了防止长期不重新提交导致该目录一直无法再次提交，已驳回超过3天的规则会允许他人再次提交。在此之前，只有您可以重新提交它。注意：判断重复是不区分大小写的

  • 不允许提交该目录

    可能由于这个目录导致许多用户出现异常，我们将其列入了 黑名单。
